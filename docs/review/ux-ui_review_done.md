## Section 1 — Критичні зауваження

### 1.1. Дубль секцій «Результати» / «Портфоліо» — ✅
- Видалено секцію «Результати, що говорять самі за себе» (круглі прев'ю) з `index.html`.
- Видалено компонент `components/results-grid/` (якщо існував).
- Залишено єдину секцію «Роботи, які надихають» (`components/portfolio/`) з прямокутними прев'ю.
- Фільтри уніфіковано: `Все · Обличчя · Масаж · Нігті · Волосся`.

### 1.2. Локація — статичне фото замість карти — ✅ (часткове, за домовленістю)
- За домовленістю з користувачем (`«картинку вставити щоб просто візуал показати»`) — інтерактивний Google Maps embed НЕ додано.
- Додано dual-deeplink кнопки «Маршрут»: `https://maps.google.com/?q=…` і `https://maps.apple.com/?q=…` (`index.html:524, 528`).
- Залишено фото фасаду + pin-маркер як візуальний placeholder.
- Підготовлено інфо-блок: адреса, телефон, графік роботи (`components/map/`).

### 1.3. Невідповідність CTA-дієслів — ✅
- Уніфіковано всі CTA на єдине дієслово `ЗАПИСАТИСЬ`:
  - шапка
  - hero
  - плаваюча кнопка
  - картки майстрів
  - картки послуг
  - мобільне меню
  - кнопка в секції локації

### 1.4. Соціальний пруф непослідовний — ⏸️ (не входило в scope)
- Залишено в backlog для наступної ітерації.
- Розширення до 8-12 відгуків — частково покрито через тестімоніали (10 карток).

### 1.5. Невідомо, куди ведуть CTA — ✅ (модалка)
- Усі CTA з атрибутом `data-booking-open` тепер відкривають booking-модалку.
- Створено компонент `components/booking-modal/` з усім UX:
  - Поля: ім'я, телефон (з паттерном `\+380\d{9}`), послуга, майстер (опц.), дата (опц.), коментар, checkbox згоди.
  - Стани форми: `idle` / `submitting` / `success` / `error` (через модифікатори `booking-modal__body--*`).
  - Валідація на сабміті: ім'я, телефон, послуга, згода — обов'язкові.
  - ARIA: `role="dialog"`, `aria-modal="true"`, `aria-labelledby`.
  - Focus trap по Tab/Shift+Tab.
  - Закриття: Esc, клік по backdrop, хрестик, `data-booking-close`.
  - Body scroll lock при відкритій модалці.
  - Submit-логіка (без бекенду): `console.log` → `setTimeout(800ms)` → `success`.

## Пост-фікси по секції Specialists (за фідбеком зі скріншота)

Користувач після раунду правок повідомив 3 баги у секції майстрів.

### S.1. Слайдер не гортається — ✅
- Файл: `js/main.js:62`
- Причина: селектор у `initCarousel(...)` був `viewport: '.specialists__cards-viewport'`, але в DOM фактично `.specialists__track` → `viewport` повертав `null` → `initCarousel` ранній `return` без бінду стрілок.
- Фікс: `viewport: '.specialists__track'`.

### S.2. Стрілки каруселі невидимі — ✅
- Файл: `components/specialists/specialists.css:148-150`
- Причина: `.carousel-arrow { background: var(--cream); border: 1px solid var(--pearl); }` — кнопка має той самий cream-фон, що й секція, і ледь видимий pearl-бордер.
- Фікс: інверсія контрасту — `background: var(--ink)`, `border: 1px solid var(--ink)`, SVG-стрілки тепер `var(--ivory)`. Додано hover: `var(--charcoal)` + `transform: scale(1.05)`.

### S.3. Кнопки «ЗАПИСАТИСЬ» приліплені до краю картки — ✅
- Файл: `components/specialists/specialists.css:141, 146`
- Причина: на десктопі `.specialist-card__btn` мала `width: 100%` (через базові мобільні стилі — кнопка тяглась від краю до краю картки) + `margin-top: 0`. Картка `padding: 32px 28px`, тому горизонтально кнопка впиралась у padding-area, а вертикально — без відступу від попереднього тексту.
- Фікс:
  - Кнопка: `display: inline-flex`, `width: auto`, `align-self: center`, `padding: 12px 32px`, `margin-top: 12px` — тепер компактна пілюля по центру.
  - Картка: `padding: 32px 28px 28px` — явний нижній відступ.
